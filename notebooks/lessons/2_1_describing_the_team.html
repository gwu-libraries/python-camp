Objectives#

  • To reflect on the utility of different kinds of data structures

  • To use Python dictionaries to capture information in a structured way

Structured data#

Like most programming languages, Python gives us various tools for organizing data, each of which involves certain tradeoffs.

In the homework you worked with the following data types:

  • floats and integers

  • strings

  • lists

The real power of these tools lies in our ability to compose them together in order to reflect both the complexity of the data we’re working with, and our goals (or those of our end users).

But it can be challenging to decide which Python types to use in a given circumstance. Before we dive into more code, take a few minutes to discuss the following questions with your team.

Questions

  1. If you’re going to the store to buy groceries, how do you generally keep track of the items you need? What pieces of information are relevant to this task?

  2. What about keeping track of the courses you’re taking in a given semester?

  3. Many of us now rely on our phones to store our contacts (friends, family, etc.). What if you had to give up your phone for a month – how would you organize your contacts? What would that look like?

  4. If you were going to record some information about the members of your team, what information would you capture? Make a list of fields that the team can agree on.

Lists versus dictionaries#

In the examples above, we were talking about lists of things. But not all lists are created equal.

For instance, in the homework, you worked with two different examples of a list:

  1. A list of book prices, where each item represented the price of a book. Beyond the price, there was no other information about the book to be gleaned from that list.

  2. A list of courses. Each string contained three pieces of information:

    • a department code

    • a course number

    • and a section number.

You used the list of course strings to create three separate lists: one for the department codes, one for the course numbers, and one for the section numbers.

But it would be handy if we could keep track of those three pieces of information in the same place (using a single data structure). Managing three lists would, in this case, be cumbersome, and if we were dealing with a large number of courses, potentially error prone.

Let’s see how Python dictionaries can help!

Tabular data#

A single dictionary is a good way to represent something in the real world – like a person, a course, or a textbook – that has multiple properties or attributes.

The dictionary’s keys can help us organize the attributes by labels, just the way we might label the different elements of each entry in an addressbook: name, email address, phone number, etc.

You might also think of the dictionary keys as the column headings in a spreadsheet. In a spreadsheet, each row uses the same column headings, which allows us to do things like sort or filter the data by particular attributes. For example, if we covert our addressbook to an Excel file, assuming we have a column for email address, we can easily compute how many of our contacts have GW email addresses.

We call such a data structure tabular because it is organized as a table, i.e., with rows and columns.

Tabular data in Python#

How do we represent tabular data in Python?

There are some very handy Python libraries for working with tables, pandas being one of the most popular. But we can also handle tabular data with just the Python types you’ve already encountered.

One common approach, which we will work through together, is to create a list of dictionaries.

  • On the analogy with a spreadsheet, each dictionary in the list would be another row of the table.

  • Each dictionary would have the same keys (with different values), corresponding to the data in the various columns of the spreadsheet.

In a similar fashion, I can add a new key/value pair to my dictionary as follows:

Saving a list of dictionaries in JSON format#

Now let’s save our database to disk. We’ll use the json library and the open function as we did on Day 1 (to load the GW Bookstore dataset), except that this time we’ll be writing to the file, not reading from it.

First we import the json library.

import json

Next we open the file for writing, and use the json.dump() method to write our list of dictionaries to the file.

In the code below, I’ve used the variable my_team to refer to the list of dictionaries created above. If you named your list something different, use that name in place of my_team.

Note that we’re saving the data to a file called team-database.json. The initial ./ indicates that this file will live in the same directory as this notebook.

with open('./team-database.json', 'w') as f:
    json.dump(my_team, f)
